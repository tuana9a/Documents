2022-01-13 why java string is immutable

string pool and "==" operator là so sánh cùng địa chỉ do vậy 2 biến trình cùng trỏ tới vùng nhớ ở spring pool sẽ bằng nhau

khi biến tring được thay đổi string pool sẽ tạo string mới và biến sẽ được trỏ lại tới vị trí này

từ đó giải thích mặc dù 2 giá trị string bằng nhau nhưng một biến khởi tạo = new String("any") khi dùng "==" sẽ sai

sử dụng string immutable cho phép nhiều biến có giá trị string giống nhau trỏ về cùng một nơi, giảm thiểu bộ nhớ sử dụng

sử dụng string immutable cho phép thread safe khi có cả ngàn thread có biến string cùng giá trị này
